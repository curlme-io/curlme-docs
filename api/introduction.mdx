## Overview

The curlme.io REST API provides programmatic access to all features:

- Manage bins (create, update, delete)
- Access captured requests
- Configure response rules
- Manage organizations
- Handle authentication

## Base URL

```
https://curlme.io/api
```

## Authentication

All API requests require authentication via the `x-api-key` header:

```bash
curl https://curlme.io/api/bins \
  -H "x-api-key: cm_your_api_key"
```

See [Authentication](/api/authentication) for details.

## Request Format

### Headers

| Header | Required | Description |
| ------ | -------- | ----------- |
| `x-api-key` | Yes | Your API key |
| `Content-Type` | For POST/PATCH | `application/json` |
| `x-bin-password` | Sometimes | Password for protected bins |

### Body

Request bodies should be JSON:

```json
{
  "name": "My Bin",
  "organizationId": "org_123"
}
```

## Response Format

All responses are JSON:

```json
{
  "id": "clx123...",
  "publicId": "abc123xyz",
  "name": "My Bin",
  "createdAt": 1706745600000
}
```

### Timestamps

Timestamps are Unix milliseconds:

```json
{
  "createdAt": 1706745600000,
  "timestamp": 1706745600000
}
```

Convert in JavaScript:
```javascript
new Date(1706745600000) // 2026-01-31T00:00:00.000Z
```

## Error Responses

Errors return appropriate HTTP status codes with a JSON body:

```json
{
  "error": "Description of the error"
}
```

### Status Codes

| Code | Meaning |
| ---- | ------- |
| `200` | Success |
| `201` | Created |
| `204` | No Content (successful deletion) |
| `400` | Bad Request (invalid input) |
| `401` | Unauthorized (invalid/missing API key) |
| `403` | Forbidden (insufficient permissions) |
| `404` | Not Found |
| `500` | Internal Server Error |

### Error Examples

**Invalid API Key:**
```json
{
  "error": "Unauthorized"
}
```

**Missing Required Field:**
```json
{
  "error": "Name is required"
}
```

**Resource Not Found:**
```json
{
  "error": "Bin not found"
}
```

**Password Required:**
```json
{
  "error": "Password required",
  "passwordRequired": true
}
```

## Rate Limits

| Plan | Requests/minute |
| ---- | --------------- |
| Free | 60 |
| Pro | 300 |
| Team | 1000 |

Rate limit info is included in response headers:

```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1706745660
```

When exceeded:
```json
{
  "error": "Rate limit exceeded"
}
```

## Pagination

List endpoints support pagination via query parameters:

| Parameter | Default | Max | Description |
| --------- | ------- | --- | ----------- |
| `limit` | 50 | 100 | Items per page |
| `offset` | 0 | - | Items to skip |

```bash
curl "https://curlme.io/api/bins/abc123/requests?limit=20&offset=40" \
  -H "x-api-key: cm_your_api_key"
```

## Filtering

Some endpoints support filtering:

```bash
# Requests since a timestamp
curl "https://curlme.io/api/bins/abc123/requests?since=1706745600000" \
  -H "x-api-key: cm_your_api_key"

# Bins for an organization
curl "https://curlme.io/api/bins?orgId=org_123" \
  -H "x-api-key: cm_your_api_key"
```

## API Endpoints

### Bins

| Method | Endpoint | Description |
| ------ | -------- | ----------- |
| `GET` | `/api/bins` | [List bins](/api/bins/list) |
| `POST` | `/api/bins` | [Create bin](/api/bins/create) |
| `GET` | `/api/bins/{binId}` | [Get bin](/api/bins/get) |
| `PATCH` | `/api/bins/{binId}` | [Update bin](/api/bins/update) |
| `DELETE` | `/api/bins/{binId}` | [Delete bin](/api/bins/delete) |

### Requests

| Method | Endpoint | Description |
| ------ | -------- | ----------- |
| `GET` | `/api/bins/{binId}/requests` | [List requests](/api/requests/list) |
| `ANY` | `/h/{binId}` | [Webhook endpoint](/api/requests/webhook) |

### Organizations

| Method | Endpoint | Description |
| ------ | -------- | ----------- |
| `GET` | `/api/organizations` | [List organizations](/api/organizations/list) |

| `GET` | `/api/organizations/{orgId}` | [Get organization](/api/organizations/get) |
| `PATCH` | `/api/organizations/{orgId}` | [Update organization](/api/organizations/update) |

### User

| Method | Endpoint | Description |
| ------ | -------- | ----------- |
| `GET` | `/api/user` | [Get current user](/api/user/me) |

### Replay

| Method | Endpoint | Description |
| ------ | -------- | ----------- |
| `POST` | `/api/replay` | Replay a request |

## SDKs & Libraries

Currently, we don't have official SDKs. The API is simple enough to use with any HTTP client:

<Tabs>
  <Tab title="JavaScript">
    ```javascript
    const response = await fetch('https://curlme.io/api/bins', {
      headers: { 'x-api-key': 'cm_your_api_key' }
    });
    const bins = await response.json();
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import requests
    
    response = requests.get(
      'https://curlme.io/api/bins',
      headers={'x-api-key': 'cm_your_api_key'}
    )
    bins = response.json()
    ```
  </Tab>
  <Tab title="cURL">
    ```bash
    curl https://curlme.io/api/bins \
      -H "x-api-key: cm_your_api_key"
    ```
  </Tab>
</Tabs>
