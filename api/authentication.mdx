## API Keys

All API requests require authentication using an API key.

### Getting an API Key

1. Sign in at [curlme.io](https://curlme.io/auth/signin)
2. Go to [Account Settings](https://curlme.io/account)
3. Click **"Create New Key"**
4. Copy the key (shown only once)

### Using the API Key

Include the key in the `x-api-key` header:

```bash
curl https://curlme.io/api/bins \
  -H "x-api-key: cm_your_api_key"
```

## Key Format

API keys follow the format:

```
cm_<64 hexadecimal characters>
```

Example:
```
cm_a1b2c3d4e5f6789012345678901234567890123456789012345678901234
```

## Error Responses

### Missing API Key

```bash
curl https://curlme.io/api/bins
```

Response `401`:
```json
{
  "error": "Unauthorized"
}
```

### Invalid API Key

```bash
curl https://curlme.io/api/bins \
  -H "x-api-key: cm_invalid_key"
```

Response `401`:
```json
{
  "error": "Unauthorized"
}
```

## Protected Bins

Some bins require a password in addition to authentication.

### Password Required Response

```json
{
  "error": "Password required",
  "passwordRequired": true
}
```

### Providing Password

Include the `x-bin-password` header:

```bash
curl https://curlme.io/api/bins/abc123 \
  -H "x-api-key: cm_your_api_key" \
  -H "x-bin-password: bin_secret_password"
```

## Security Best Practices

<CardGroup cols={2}>
  <Card title="Use Environment Variables" icon="leaf">
    Never hardcode API keys in your source code.
  </Card>
  <Card title="Separate Keys" icon="key">
    Use different keys for development, staging, and production.
  </Card>
  <Card title="Rotate Regularly" icon="rotate">
    Create new keys periodically and revoke old ones.
  </Card>
  <Card title="Minimal Scope" icon="shield">
    Use keys only where needed; avoid sharing.
  </Card>
</CardGroup>

### Environment Variables

```bash
# Set in your environment
export CURLME_API_KEY=cm_your_api_key
```

```javascript
// Use in code
const apiKey = process.env.CURLME_API_KEY;
```

### .gitignore

Ensure API keys aren't committed:

```gitignore
.env
.env.local
.env.*.local
```
